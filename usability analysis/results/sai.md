# Summary
I've been using KubeVirt for several months and I like it.
Specifically, the cloud native approach where everything is API accessible, which makes it easier to build tooling around and the community has done a great job of that as expected. Due to the fact it's APIs first, it opens up many approaches to manage virtual machines.

# Basic
Personally, from having worked with older virtualization platforms and Kubernetes, I'd prefer to use KubeVirt as my virtualization platform.
It is primarily because I'm already storing all my Kubernetes manifest in Git and have the CI setup for that, so why not also create virtual machines in a similar way.

#
I'd say I like this relatively new take on how to deal with virtual machines. At the same time, it may not be for everyone. In the end, it depends on what problems you are trying to solve and the need.
